

## Проблема

Сейчас в проекте две **несвязанные** системы прогнозов:

1. **Публичная страница** `/predictions` читает прогнозы из статичного TypeScript-файла `src/data/predictions.ts` (34 прогноза, зашиты в код).
2. **Админка** `/admin/predictions` читает и пишет в таблицу базы данных `predictions` — а эта таблица **пустая** (0 записей).

Поэтому в админке ничего не видно, а изменения, которые вы делаете, никак не влияют на публичный сайт. Это и есть корень проблемы.

## Решение

Сделать **базу данных единственным источником правды** для прогнозов. Админка пишет в БД → публичная страница читает из БД → всё синхронизировано в реальном времени.

```text
ДО (сломано):                     ПОСЛЕ (правильно):

[Статичный .ts] → /predictions    [БД predictions] ← Админка
[БД (пусто)]   ← Админка                ↓
                                  /predictions (читает из БД)
```

## Шаги реализации

### 1. Перенос 34 прогнозов из кода в базу данных
Одноразовая миграция: преобразую массив `predictions` из `src/data/predictions.ts` в SQL `INSERT` и заливаю все 34 записи в таблицу `predictions`. Доказательства (`evidence`) при этом раскладываются в связанную таблицу `prediction_evidence`. После заливки в админке вы сразу увидите все 34 прогноза.

### 2. Переключение публичной страницы на чтение из БД
Страница `/predictions` и компонент `PredictionsTimeline` сейчас импортируют `predictions` из статичного файла. Меняю это на загрузку через TanStack Router loader, который тянет данные из таблиц `predictions` + `prediction_evidence` через Supabase. Структура данных (`Prediction`, `Evidence`, `getStats`) остаётся прежней — компоненты работают без изменений.

### 3. Realtime-синхронизация
Включаю Supabase Realtime для таблиц `predictions` и `prediction_evidence`. Это значит, что когда вы в админке нажимаете «Сохранить», изменения **автоматически** появляются на публичной странице у всех открытых посетителей — без перезагрузки.

### 4. Чистка
- Файл `src/data/predictions.ts`: оставляю только типы (`Prediction`, `PredictionStatus`, `CATEGORIES`, `STATUS_LABELS`, `STATUS_COLORS`, `SOURCE_LABELS`, `getStats`) — массив с данными удаляется.
- API-эндпоинт `/api/predictions/json` тоже переводится на чтение из БД (чтобы внешние интеграции получали актуальные данные).
- Файл `src/data/predictions.validation.ts` и баннер валидации — обновляются под новую модель загрузки.

### 5. Улучшения админки
В процессе перевода добавлю:
- **Загрузка в реальном времени**: список в админке тоже подписан на Realtime — если вы откроете админку с двух устройств, изменения мгновенно видны на обоих.
- **Счётчики статусов** наверху админки (Сбылось / Частично / В процессе / …) — чтобы сразу видеть общую картину.
- **Поиск и фильтр** по статусу/категории — с 34+ прогнозами без этого неудобно.

## Технические детали

**Файлы, которые изменяются:**
- `supabase/migrations/<новая>.sql` — INSERT 34 прогнозов + всех их evidence + `ALTER PUBLICATION supabase_realtime ADD TABLE …` для realtime.
- `src/data/predictions.ts` — удаляется массив, остаются только типы и `getStats`.
- `src/data/predictions.fetch.ts` (новый) — функция `fetchPredictions()`, которая тянет из Supabase и собирает в формате `Prediction[]`.
- `src/routes/predictions.tsx` — добавляется loader, читает через TanStack Query + Realtime-подписку.
- `src/components/PredictionsTimeline.tsx` — принимает `predictions` пропсом вместо импорта (или читает через хук).
- `src/components/PredictionsValidationBanner.tsx` — то же самое.
- `src/routes/api.predictions.json.ts` — server handler читает из БД.
- `src/routes/admin.predictions.tsx` — добавляется realtime-подписка, фильтры, счётчики.

**RLS policies уже корректные:** публичное чтение открыто (`Predictions are publicly readable`), запись только админам (`has_role(auth.uid(), 'admin')`). Менять не нужно.

**Что у вас останется работать как раньше:**
- Внешний вид публичной страницы `/predictions` не меняется.
- Все 34 прогноза с их evidence сохраняются.
- Админка по адресу `/admin/predictions` — структура та же, но теперь видит реальные данные.

## Результат

После внедрения:
- Заходите в `/admin/predictions` → видите все 34 прогноза.
- Меняете статус, добавляете evidence, создаёте новый прогноз → нажимаете «Сохранить».
- Открываете `/predictions` (или у вас она уже открыта) → изменения **уже там**.

Это и есть полноценный CMS-режим для администратора, о котором вы говорите. По той же схеме потом подключим публикации, блог и остальные разделы.
