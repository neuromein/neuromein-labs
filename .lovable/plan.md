# План: управление выступлениями и обучениями в ЛК

## Что получится

В личном кабинете появится новый раздел **«Выступления»** — там вы сможете добавлять, редактировать, скрывать и удалять карточки выступлений и обучений. Загруженные фото будут храниться в Lovable Cloud (без копирования URL вручную). Каждое выступление получит отдельную страницу на сайте с большим фото и подробным описанием.

## Структура карточки

Каждая карточка будет содержать:
- **Фото** (загрузка из ЛК)
- **Организация** (заголовок, например «Центральный Банк Армении»)
- **Роль** (бейдж «Спикер-эксперт» и т.п.)
- **Краткое описание** (для карточки в слайдере)
- **Длинное описание** (для отдельной страницы)
- **Дата события**
- **Место проведения** (опционально)
- **Ссылка на материал/видео** (опционально)
- Системные: порядок отображения, видимость (вкл/выкл)

## Где это появится на сайте

1. **Главная** — слайдер «Выступления и обучения» (как сейчас, но данные из БД). Карточки кликабельны.
2. **Отдельная страница `/speaking/[slug]`** — большое фото, дата, место, длинное описание, кнопка «Смотреть материал» (если указана ссылка).
3. **Личный кабинет** — пункт «Выступления» в боковом меню рядом с «Прогнозы» и «Публикации».

## Технические детали

### База данных
Новая таблица `speaking_engagements`:
- `id`, `slug` (уникальный, для URL)
- `organization`, `role`, `caption`, `description`
- `image_url` (ссылка на файл в Storage)
- `event_date` (date), `location` (text, nullable), `external_url` (text, nullable)
- `display_order`, `is_visible`, `created_at`, `updated_at`

RLS: публичное чтение видимых записей, запись/обновление/удаление — только для `admin` (как у `publications`).

### Хранилище фото
Создам публичный bucket `speaking-images` с RLS:
- Чтение: всем
- Загрузка/удаление: только `admin`

В админке — стандартный input `type="file"`, после выбора файл загружается через `supabase.storage`, в БД сохраняется публичный URL.

### Миграция данных
Существующие 6 карточек из `SpeakingSlider.tsx` (МГУ, R1, ИИ-Банкинг, ЦБ Армении, Yandex, МАИ) перенесу в БД insert-ом, изображения из `src/assets/speaking/` оставлю на месте и пропишу как локальные пути в `image_url`. После переноса вы сможете их редактировать или заменить через ЛК.

### Файлы

**Новые:**
- `supabase/migrations/...` — таблица + RLS + bucket
- `src/data/speaking.fetch.ts` — загрузка списка / по slug
- `src/routes/admin.speaking.tsx` — CRUD-интерфейс (как `admin.publications.tsx`)
- `src/routes/speaking.$slug.tsx` — публичная страница выступления

**Изменю:**
- `src/components/SpeakingSlider.tsx` — данные из БД, карточки → ссылки на `/speaking/$slug`
- `src/routes/admin.tsx` — добавлю пункт «Выступления» в навигацию
- `src/routes/index.tsx` — передача данных в слайдер (loader)
